**Decisions Made:**

1. BrowserController/BrowserWindow split — separates logic from UI; BrowserDaemon tracks controllers not windows
2. Sequential IDs instead of screen-keyed registry — allows multiple windows per screen
3. Counter resets on daemon restart — simple, no persistence needed
4. Cap at 100 windows — anything over is an error condition worth investigating
5. Always open new window — no "update in place"; each command = new browser
6. QLocalSocket import removed — unused after stdlib socket approach
